Generator Functions

Definition

These are functions which implement iterators.These functions use a yield statement instead of a return statement.
The yield statement tells the interpreter to store local variables and record the current position in the generator.
This helps to resume from that stored location when the next call is made to the generator.

Syntax

def generator_name():
yield x # expression to be evaluated when next() is called on a generator

Example Function

Let's make a generator that generates numbers from 1 to 9.

def num_generator():
for j in range(10):
yield j

a = num_generator()
for i in a: # we can also iterate a generator using a for loop just like a list
print(i)

Benefits of using Generators

These use low memory as these generate on demand values.As values are lazily generated we can use any value without having to wait for all the values.
There comes a disadvantage with generators and that is we can use the set of generated values only once.
